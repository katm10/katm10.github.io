Final Project


  • GeneticAlgo: This class contains all of the code to take the best schedules, cross them over, mutate them, and create new generations of schedules.
  • Dept: This class contains academic departments (ie. Math, Social Studies, English, Science, etc) which each contain the corresponding courses that fit within them.
  • Data: This class initializes all of the Dept, Course, Student, and Teacher data to be used in the genetic algorithm.
  • Driver:This calls the Data, Population, GeneticAlgo, Scheduler, and Student classes to allow the user to interact with the programs.
  • Population: This creates lists of Schedulers and sorts them based on their fitness.
  • Scheduler: This class contains the code to randomly create a schedule, calculate its fitness, print schedules, and interact with the Student class.
  • Course: This class contains all of the information about a course, like its name, id, and the teachers who can teach it.
  • Class: This is a subclass of Course. It contains the more specific information about a class, like the period, the teacher, and which students are enrolled.
  • Student: This contains information on individual students, like their name, grade, GPA, courses they want, and classes they are enrolled in.
  • Person: This is an interface which both Student and Teacher extend.
  • Teacher: This contains information on individual teachers, like their name and id.

Check out the code on Github: github.com/katm10/APCS-Final I wanted to challenge myself and learn some new concepts for the Spring semester project, so I learned how to use genetic algorithms and incorporate them into my project from last semester. However, it was pretty difficult, so the fitness only improves a tiny bit over time, and as you can see, the output is not perfect and has many impossible scenarios. The way that the program works it that it creates an initial schedule by just randomly choosing a teacher, a period, and students for each class using the Math.random() function. Then, after initializing several of these, it calculates each of their fitnesses based on how many conflicts there are, like having a teacher or student have two classes during the same period or not giving a student enough classes to graduate. The best ones are then "cross over" and the program uses Math.random() once again to choose the schedule that will move on. This one is then mutated by changing one of its classes, but I think this is where I messed up since there are few mutations from the initial to the final product.
